Lead Scraper — Setup Guide

Get the weekly lead scraper running on your own Mac, start to finish.

Be the name they know. — Forward is the only way. We forge on.

This walks you through everything, click by click. No coding required — you'll copy a few things and click some buttons. By the end you'll have a Forge Leads icon on your Desktop that pulls fresh, scored leads in your region whenever you want them.

⏱ Plan for about 45–75 minutes
👋Two things Teo sets up for you first. Before you start, make sure Teo has (1) added you as a collaborator on the GitHub code repository (you'll get an email invite — accept it), and (2) is free for ~20 minutes while you do Part 4 (the Google setup). That part is the fiddly one, and having him on a quick screen-share makes it painless. Everything else you can do solo.

The big picture (what each part does)

  • Part 1 — Install two free tools (Python & GitHub Desktop).
  • Part 2 — Download the code to your Mac.
  • Part 3 — Run the one-click installer (builds everything + your Desktop icon).
  • Part 4 — Get your Google keys (the longer part — Teo on standby).
  • Part 5 — Plug your keys in.
  • Part 6 — First run: sign into Google, watch it work.
  • Part 7 — How to run it every week.

1Install two free tools

You need two free apps. Install both, then come back.

A) Python 3.12

  1. Go to python.org/downloads/release/python-3120.
  2. Scroll to the bottom, under Files, click “macOS 64-bit universal2 installer” to download the .pkg file.
  3. Open the downloaded file and click through Continue → Continue → Agree → Install (enter your Mac password if asked).

B) GitHub Desktop

  1. Go to desktop.github.com and click Download for macOS.
  2. Open the downloaded file and drag GitHub Desktop into your Applications folder.
  3. Open GitHub Desktop and click Sign in to GitHub.com. Log in with your GitHub account (the same email Teo invited).
💡What's GitHub? It's just where the code lives online — like a shared Google Drive, but for software. GitHub Desktop is the friendly app that copies it to your Mac.

2Download the code

  1. First, accept Teo's invite: check your email for an invitation to the forge-local-lead-scraper repository and click Accept. (If you don't see it, ask Teo to re-send it.)
  2. In GitHub Desktop, go to File → Clone repository.
  3. Pick the forge-local-lead-scraper repo from the list. For Local path, the default is fine — but note where it's putting the folder (usually Documents/GitHub/forge-local-lead-scraper). Click Clone.
  4. When it finishes, click Repository → Show in Finder. A folder window opens — keep it open, you'll use it next.
💡“Clone” just means “download your own copy.” That folder on your Mac is now yours — your keys and leads stay on your machine.

3Run the one-click installer

This builds everything automatically and puts the Forge Leads icon on your Desktop. It's safe to run more than once.

  1. In the folder from Part 2, find the file named Setup Forge Local.command and double-click it. A black Terminal window opens and starts working.
  2. If macOS blocks it with “cannot be opened because it is from an unidentified developer,” right-click the file → Open → Open in the popup. (You only do this once.)
  3. Watch it run through steps 1–4. It checks Python, installs what it needs, and creates your Desktop icon. When you see “🎉 Setup is done,” you can read the two reminders and close the window.
⚠️If it says “Python 3.12 is not installed”, go back and finish Part 1A, then double-click Setup again.

You'll now have a Forge Leads icon on your Desktop. It won't fully work yet — you still need your Google keys (next).

4Get your Google keys

This is the longest part. Take it slow. This is the best time to have Teo on a screen-share.

You're getting two things from Google: an API key (lets the tool look up businesses) and an OAuth client file (lets it create your Google Sheets and email your report). Both are free.

4.1 — Create your project

  1. Go to console.cloud.google.com and sign in with your work Google account.
  2. At the very top, click the project dropdown (says “Select a project”) → New Project.
  3. Name it Forge Local Leads → Create. Wait a few seconds, then make sure that project is selected in the top dropdown.

4.2 — Turn on the Places API

  1. In the search bar at the top, type Places API (New) and click it in the results.
  2. Click the blue Enable button. (If it asks you to enable billing first, do Step 4.3 below, then come back and Enable.)

4.3 — Turn on billing (required, but you won't be charged for normal use)

⚠️Google requires a credit card on file to use the maps data — but normal weekly scraping stays inside Google's $200/month free credit. A typical week costs a few cents. You will not be charged for using this the way we use it.
  1. In the top search bar, type Billing and open it.
  2. Click Link a billing account → Create billing account, and follow the prompts to enter your card details.
  3. Make sure the billing account is linked to your Forge Local Leads project.

4.4 — Create your API key

  1. Top search bar → type Credentials → open the APIs & Services → Credentials page.
  2. Click + Create Credentials → API key.
  3. A box pops up with your new key. Click Copy and paste it somewhere safe for a minute (a sticky note or Notes app). It looks like AIzaSy…
  4. (Recommended) Click Edit API key → under API restrictions choose Restrict key → check Places API (New) → Save. This keeps the key safe.

4.5 — Set up the consent screen — choose “Internal”

⚠️First confirm you're signed into Google Cloud with your @forgelocal.app account (top-right) — not a personal Gmail. “Internal” only works with your company account.
  1. Top search bar → OAuth consent screen (newer console: “Google Auth Platform” → “Audience”) → open it.
  2. For User type / Audience, choose Internal → Create.
  3. Fill in App name (Forge Local), User support email (yours), and Developer contact email (yours), then Save. You can skip “Scopes,” and with Internal there's no Test users step.
💡Why Internal (this matters): any @forgelocal.app account can sign in, there's no weekly re-auth, and you skip Google's verification process entirely. Picking “External” instead is what causes the weekly lockout — so use Internal.
⚠️If Internal is greyed out, you're either signed in with a personal account or the project isn't under the company org. Sign in with your @forgelocal.app account and retry, or ping Teo.

4.6 — Create & download the OAuth client file

  1. Go back to Credentials (top search bar → Credentials).
  2. Click + Create Credentials → OAuth client ID.
  3. For Application type, choose Desktop app. Name it Forge Local Desktop → Create.
  4. In the popup, click Download JSON. A file lands in your Downloads folder (named something like client_secret_…json).
  5. Rename that file to exactly oauth-client.json (keep the .json ending). You'll move it in the next part.

5Plug your keys in

Two quick drops into your project folder (the one from Part 2 — reopen it in Finder if you closed it).

5.1 — Drop in your OAuth file

  1. In your project folder, open the credentials folder.
  2. Drag the oauth-client.json file you just renamed (from Downloads) into that credentials folder.

5.2 — Add your API key & email

  1. In the project folder, find the file named .env (just “.env”, nothing before the dot). Right-click it → Open With → TextEdit.
  2. Find the line GOOGLE_MAPS_API_KEY= and paste your API key right after the = (no spaces, no quotes).
  3. Find the line REPORT_EMAIL= and type your work email right after the =.
  4. Save (⌘ + S) and close TextEdit. Leave the other lines exactly as they are.
GOOGLE_MAPS_API_KEY=AIzaSyYourKeyHere REPORT_EMAIL=neema@forgelocal.app
⚠️Can't see the .env file? In Finder press ⌘ + Shift + . (period) to show hidden files. Press it again later to re-hide them.

6First run — sign in and watch it work

  1. Double-click the Forge Leads icon on your Desktop. A Terminal window opens with a menu.
  2. If macOS blocks it (“unidentified developer”), open System Settings → Privacy & Security, scroll down, and click Open Anyway. (One time only.)
  3. At the menu, type 2 (Configure today's run) to pick your state, niche, and how many leads — or type 1 to run with the defaults. Press Return.
  4. The first time only: your web browser pops open asking you to sign into Google. Choose your account → if you see a “Google hasn't verified this app” warning, click Continue (it's your own app) → click Allow. The browser will say “authentication flow has completed” — close it.
  5. Back in Terminal, watch it scrape → find emails → score → build your Google Sheet → email you the report. It prints a Sheet link at the end.
✅That's it — you're live. Open the Sheet link, or check your inbox for the report. Leads are color-coded by how much they need us (🔴 hottest first), with phone and email where we could find one.
📧About the Email column: roughly 1 in 3 businesses with a website will have an email we can pull — the rest you reach by phone. A blank email is normal and honest, not a bug. We never guess or make up addresses.

7Running it every week

  1. Double-click Forge Leads on your Desktop.
  2. Type 2 to set this week's region/niche (or 1 to reuse last week's), press Return.
  3. Let it run and open your Sheet. With the Internal setup, you usually won't be asked to sign in again.
🔁No weekly sign-in. Because the app is Internal (not “Testing”), your login sticks — you shouldn't get logged out every week. If Google ever does ask, just pick your @forgelocal.app account and Allow — one click, done.

🛠 Troubleshooting

What you seeWhat to do
“cannot be opened — unidentified developer”Right-click the file → Open → Open. For the Desktop icon: System Settings → Privacy & Security → Open Anyway. One time each.
“command not found: python”You're typing in the wrong place — just use the Forge Leads Desktop icon, which handles this for you.
Browser keeps asking me to sign inWith the Internal setup this is rare — pick your @forgelocal.app account and Allow. If it happens on every run, your app is likely set to "External/Testing" instead of Internal — recheck Part 4.5.
“GOOGLE_MAPS_API_KEY not found”Your .env is missing the key, or wasn't saved. Redo Part 5.2 and make sure you pressed Save.
API error about billing / “REQUEST_DENIED”Billing isn't linked or Places API (New) isn't enabled. Recheck Parts 4.2 and 4.3.
Lots of blank emailsExpected — only ~1 in 3 sites expose an email. Use the phone number for the rest.
My state or niche isn't in the listThe menu only shows regions/niches set up in the config. Ask Teo to add yours — it's a one-line change on his end.
Anything elseScreenshot it and send it to Teo. Don't guess — a 2-minute message saves an hour.
